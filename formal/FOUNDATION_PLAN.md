> arbitrary environments) and the crypto-jet law part of steps 7-8. Current
> state: `README.md`; extraction plan: `GENERALIZATION.md`.

This directory currently proves the multisig security property at the covenant
model boundary. The next step is a bridge proof from the concrete Simplicity
program and Elements environment into that model.

See `BRIDGE_RESEARCH.md` for the detailed compiled-program bridge analysis and
trust-boundary options.

Current checked bridge artifacts:

- `ElementsJets.v`: multisig-only Elements jet subset and decoder.
- `ElementsJetSemantics.v`: explicit semantic laws for the whitelisted jet
  interface plus checked bridges from successful participant, threshold, and
  minimum-input assertions to source/model premises. It also relates the
  Elements environment accessors to the abstract transaction lists used by the
  model.
- `SimplicityByteDecoder.v`: executable byte-to-structural-program decoder,
  C-compatible natural-number bound, deployed DAG length bound, CMR checker,
  strict raw backreference proofs, close-bitstream padding check, jet
  whitelist, and structural DAG facts.
- `ElementsJetCmr.v`: the consensus CMR byte table for the whitelisted
  Elements jets, copied from the upstream Elements `primitiveJetNode.inc` table
  into Coq definitions with checked byte-range and 256-bit-length proofs.
- `CmrWellFormed.v`: the 256-bit-output contract for CMR algebras, plus
  checked/unchecked CMR agreement and Elements jet CMR adapter preservation
  theorems.
- `FoundationCmrAlgebra.v`: a foundation-shaped CMR adapter interface matching
  the upstream Merkle-root tag/`compress_half`/`compress` structure, with
  audited shape and well-formedness theorems for the local checked-CMR decoder.
  The shape lemmas make explicit that assertion nodes share the case tag,
  two-child disconnects use the disconnect-tag unary CMR, and Elements jets come
  from the checked Elements CMR table.
- `TypedBridge.v`: executable structural type-table checker, parameterized by
  hooks for jets, witnesses, words, fails, and disconnect forms.
- `ElementsJetTypes.v`: the upstream source/target type table for the
  whitelisted Elements jets plus the Simplicity word type rule, fail rejection,
  and two-child disconnect type rule, represented compositionally as
  `BridgeType` values and plugged into `TypedBridge.v`'s hooks.
- `MultisigCertificate.v`: decode-only and CMR-checked byte-certificate
  checkers plus aggregate byte and typed bridge evidence theorems.
- `CompiledMultisigExample.v`: the concrete generated byte certificate accepted
  by the audit. It exposes the decoded threshold and three declared
  participants, proves their static source/model checks, and composes those
  artifact facts with explicit prefix and counted-vote premises to derive
  `multisig_covenant_succeeds` for the concrete participant set. It also
  composes that result with the model security theorem to derive the concrete
  artifact's threshold-distinct-declared-participant authorization claim and
  the full model security property under the same dynamic premises.
- `MultisigTypedCertificate.v`: a typed byte-certificate checker that combines
  the streaming CMR-checked byte certificate with an exported per-node type
  table using the checked Elements jet type hook, plus composition theorems from
  decode evidence and ordinary CMR equality to compact typed checked-CMR bridge
  evidence, and projections from compact typed checked-CMR bridge evidence back
  to byte evidence, decoded bytes, and checked CMR equality. It also proves that
  atom-free compact type definitions expand to atom-free type tables and root
  arrows.
- `BridgeTypeTranslation.v`: a Unit/Sum/Prod type-algebra translation interface
  for atom-free bridge types, plus a concrete theorem that the compiled typed
  certificate translates for every such algebra.
- `FoundationTypes.v`: an audited adapter from the atom-free bridge type
  certificate to the actual upstream `Simplicity.Ty` constructors.
- `FoundationCore.v`: an audited adapter from checked typed evidence for core
  structural node forms to actual upstream `Simplicity.Core.Term` witnesses,
  conditional on child terms, plus recursive typed-prefix child-term provider
  construction and typed-byte root composition conditional on explicit non-core
  primitive term providers.
- `FoundationElementsProviders.v`: a narrowed provider interface for the
  multisig typed-certificate hook profile. It exposes only assertion, Elements
  jet, witness, word, and two-child disconnect term-provider obligations, while
  proving fail nodes and reserved one-child disconnect nodes cannot satisfy the
  hooks.
- `CompiledMultisigFoundation.v`: an audited instantiation of the generic
  typed-byte root composition theorem for the actual generated compiled
  multisig typed certificate, conditional on either the generic non-core
  provider or the narrowed Elements provider family. It also instantiates the
  root-term theorem from both the compact typed checked-CMR bridge evidence path
  and the direct successful typed streaming checked-program path for the
  concrete artifact.
- `CompiledMultisigFoundationCmrEvidence.v`: audited byte-decoder projections
  from the direct successful typed streaming checked-program path specialized to
  `foundation_elements_cmr_algebra ops`. It exposes the streamed decoded-program
  equality and the checked-CMR equality to the exported artifact CMR.
- `CompiledMultisigFoundationSecurity.v`: composes the checked typed+CMR
  root-term path with the concrete artifact security theorem under explicit
  dynamic prefix/`CountVotes`/threshold-count premises, and under the stronger
  semantic-side package of static/prefix/minimum Elements assertion-success
  facts plus executed vote slots and the final vote-threshold assertion. The
  strongest variants now start from a successful typed streaming checked-program
  run rather than separate decoded-program and unchecked-CMR premises.
- `CompiledMultisigFoundationCmrSecurity.v`: specializes the strongest checked
  security theorem to `foundation_elements_cmr_algebra ops`, removing the
  arbitrary-CMR-algebra surface from the top checked artifact theorem while
  keeping the successful checked-program run as an explicit premise.
- `MultisigSourceBlocks.v`: source-block lemmas for participant uniqueness,
  static threshold bounds, and a composition theorem from named source-block
  premises to `multisig_covenant_succeeds`.

## Current proof boundary

`MultisigSecurity.v` abstracts:

- 32-byte hashes, public keys, and signatures,
- BIP-0340 verification,
- SHA256 over covenant words,
- the vote Taproot script hash commitment, and
- transaction introspection over input script hashes, input hashes, and output
  hashes.

Under those abstractions, `multisig_success_security_property` proves that a
successful multisig covenant has at least `threshold` counted vote slots whose
participants are distinct. Each counted vote is from a declared participant
slot, signs
`SHA256(vote_executable_leaf_hash || base_message)`, and has an input script hash
matching the vote Taproot commitment for the same leaf hash and signature.

`multisig_success_authorizes_threshold_distinct_declared_participants` exposes
the distinct-participant authorization claim separately.

The missing foundation layer for this contract is an Elements/Taproot primitive
specification: transaction introspection jets, Schnorr verification, Taproot
script tree hashing, and the exact SHA256 word concatenations used by the vote
message and base message. The current local `TypedBridge.v` mirrors the shape
of C type inference, `ElementsJetTypes.v` instantiates the whitelisted jet
source/target hook from the upstream Elements type table, and
`MultisigTypedCertificate.v` composes that type-table checker with the
streaming CMR-checked byte certificate. Word nodes are checked locally as
`Unit -> TWO^(2^n)` with a compact bitstring of length `2^n`, fail nodes are
rejected by the standard typed hook profile, and two-child disconnect nodes are
checked against the delegation type shape. The Rust exporter now emits a
compact indexed per-node type artifact and expected root-arrow index through
`coq-typed`; `MultisigTypedCertificate.v` expands that artifact before invoking
the typed checker. The typed checker now proves the table length, real/hidden
node-entry shape, indexed per-node accepted-form typing premises, and
real-root-arrow facts that the later foundation conversion needs; composed with
byte child-reference evidence, it also proves that every child reference has a
typed arrow and that no typed byte artifact node is `SFail` or `SDisconnect1`.
The concrete compiled typed certificate is also proved atom-free, so its
checked type shape contains only Unit/Sum/Prod bridge constructors, and it is
proved to translate into any Unit/Sum/Prod type algebra. `FoundationTypes.v`
instantiates that interface with upstream `Simplicity.Ty`. `FoundationCore.v`
proves the pure core node-form step into upstream `Simplicity.Core.Term` and
recursively builds child-term providers from checked typed prefixes, then
packages typed-byte root term construction behind explicit non-core primitive
providers. `FoundationElementsProviders.v` narrows that obligation to the
allowed Elements non-core node families and discharges rejected fail and
`disconnect1` forms from the hook proofs. The next term-construction layer
should not invent a separate Simplicity language: it should extend or parallel
`FoundationCore.v` with a target based on the upstream
`FullSimplicityWithDelegation.Algebra` interface from `Simplicity.Primitive`
and `Simplicity.Delegation`, so assertions, witnesses, Elements jets, words,
and two-child disconnects are represented by the same algebraic constructors
used by the foundation CMR and semantic infrastructure.
`CompiledMultisigFoundation.v` applies
the generic theorem to the actual generated multisig compact typed certificate,
including the checked-CMR compact typed bridge evidence path and variants that
consume the narrowed Elements provider family. `ElementsJetSemantics.v` now starts the semantic side
by proving that the relevant successful static, prefix/current-index,
environment lookup, and minimum-input jet assertions imply the corresponding
source/model premises. `ElementsJetEnvironment.v` now composes those premises
with `ElementsVoteSlotsExecution` and the final vote-threshold assertion to
derive `CountVotes`, the threshold-count fact, and the actual
`multisig_covenant_succeeds` model predicate. `CompiledMultisigExample.v`
specializes that source/model composition to the concrete decoded artifact's
threshold and participant keys for the static side, and then composes it with
the existing model security theorem to derive the concrete artifact's
authorization/security property from the same dynamic premises.
`CompiledMultisigFoundationSecurity.v` now composes that security theorem with
the checked typed+CMR foundation root-term entry point. The strongest current
statement starts from a successful typed streaming checked-program run over the
concrete byte/type/CMR artifact and narrowed Elements term providers, consumes
semantic static/prefix/minimum assertion-success facts, an executed vote-slot
trace, and the final threshold assertion. `FoundationCmrAlgebra.v` now narrows
that successful checked-program premise to a foundation-shaped CMR adapter
surface, and `CompiledMultisigFoundationCmrSecurity.v` exposes the corresponding
top theorem. The remaining CMR bridge is to instantiate `FoundationCmrOps` from
upstream `Simplicity.Digest`/`Simplicity.MerkleRoot` and prove/run the concrete
checked-program equality for the deployed artifact. The remaining type/term
bridge is to implement concrete foundation providers for assertion, Elements
jet, witness, word, and two-child disconnect primitives, connect witness values
during semantic evaluation, and complete the hash/Taproot/signature execution
bridge that produces the prefix/static assertion facts and the vote execution
relation.

## Bridge theorem target

The bridge proof should reduce the concrete multisig program to the model:

```coq
Theorem compiled_multisig_implies_model_success :
  forall env tx current_script_hash total_proposed_outputs
         threshold current_index participants votes,
    (* concrete Simplicity/primitive evaluation succeeds *) ->
    (* env decodes to tx/current_index/participants/votes/constants *) ->
    multisig_covenant_succeeds
      tx current_script_hash total_proposed_outputs
      threshold current_index participants votes.
```

Composing that theorem with `multisig_success_authorizes_threshold` yields the
end-to-end security statement. The bridge proof must account for the concrete
SIMF pairwise participant inequality assertions and derive `NoDup participants`
for the model predicate.

The stronger final statement should be phrased against the compiled artifact,
not against a hand-written source term.  In outline:

```coq
Theorem compiled_multisig_artifact_security :
  forall env witness_values tx current_script_hash
         total_proposed_outputs threshold current_index participants votes,
    (* Coq accepted the exported byte/type/CMR certificate. *) ->
    (* The accepted artifact denotes a foundation Simplicity term. *) ->
    (* That term successfully evaluates under the Elements env and witnesses. *) ->
    (* The env/witness decoding corresponds to tx/participants/votes. *) ->
    exists counted,
      threshold <= length counted /\
      NoDup (map counted_participant counted) /\
      Forall (fun cv => In (counted_participant cv) participants) counted /\
      Forall (counted_vote_valid tx) counted.
```

The proof may still be decomposed through
`compiled_multisig_implies_model_success`; the important trust-boundary rule is
that the left side starts from the checked compiled byte/type/CMR certificate
and foundation evaluation, not from an assumed SIMF source semantics.

## Implementation sequence

1. Fix the foundation dependency shell on this host: allow the upstream
   unsupported CompCert package or choose a supported platform, replace the
   VST SHA derivation's hardcoded `gcc` post-build command with an available C
   compiler, and build the foundation Coq project in one Coq version.
2. Build this `formal/` project in the same Coq version and add the foundation
   Coq project as a dependency of `formal/_CoqProject`.
3. Instantiate `FoundationCmrOps` from `Simplicity.MerkleRoot` for core nodes,
   assertions, witnesses, fails, and words; use `ElementsJetCmr.v` for the
   multisig Elements jet CMRs, and discharge the concrete dependency/toolchain
   side of the adapter. Then prove that the typed streaming checked-program run
   with `foundation_elements_cmr_algebra ops` accepts the deployed artifact's
   exported CMR.
4. Tie `ElementsJetTypes.v`'s checked source/target table to the foundation type
   representation. The compact indexed type artifact from `coq-typed` is
   already checked, proved atom-free, and proved translatable through the
   Unit/Sum/Prod type-algebra interface for the concrete decoded program in the
   default audit.
5. Extend the current core-only term bridge to the foundation's richer
   algebraic term representation. The target should use upstream
   full-Simplicity-with-delegation algebraic terms: core combinators from the
   existing adapter, assertions from `Assertion`, witnesses from `Witness`,
   words through foundation `scribe`/word values, jets through `Jet.jet`, and
   two-child disconnects through `Delegation.disconnect`.
6. Define an Elements environment record for the contract inputs and outputs, or
   extend the existing Bitcoin primitive environment if that proves less invasive.
7. Specify each contract jet used by the multisig and vote covenants. The
   static participant/threshold/prefix/current-index/minimum-input assertion
   laws, the environment-to-transaction-list relation, and the concrete
   artifact's static threshold/participant checks are now captured and composed
   to the model predicate when executed vote-slot evidence and the final
   threshold assertion are supplied. The remaining concrete specs are SHA256 word
   hashing, Schnorr verification, Taproot script hash construction, witness
   decoding, and proving the loop bodies produce the prefix/static assertion
   facts and vote execution relation from foundation evaluation.
8. Prove each jet specification refines the abstract functions used in
   `MultisigSecurity.v`.
9. Represent the generated multisig Simplicity term in Coq, preferably by
   exporting a stable Coq AST from the Rust/SIMF builder instead of hand-copying
   the term.
10. Prove `compiled_multisig_implies_model_success` by symbolic evaluation of the
   term against the jet specs.
11. Compose the bridge theorem with the checked model theorem for distinct
   declared participants.
